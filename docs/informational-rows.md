# KB 결과의 안내성 행 처리

KB 빠른조회 결과 테이블에는 실제 거래와 같은 8셀 본거래 행 및 1셀 상세행 구조를 사용하지만,
출금액과 입금액이 모두 0이고 거래유형 셀이 공백인 안내성 행이 포함될 수 있다.

다음 조건이 모두 검증된 행은 거래가 아닌 `informational row`로 분류한다.

- 출금액과 입금액 원문이 정상적으로 숫자 변환되어 모두 0이다.
- 거래유형 원문은 trim 후 빈 문자열이다.
- 본거래 행은 예상한 8셀 구조와 열 매핑을 유지한다.
- 각 본거래 행 바로 뒤에 정상적인 1셀 상세행이 정확히 하나 존재한다.
- 날짜, 잔액 및 다른 필수 필드의 정규화와 검증이 성공한다.

안내성 행은 `sourceKey` 생성, 중복 비교 및 Google Sheets append 대상에서 제외한다. 제외 시
`informational_row_skipped` 이벤트와 행 인덱스, `ZERO_AMOUNT_EMPTY_TRANSACTION_TYPE` 사유만 기록하며
셀 원문이나 거래 데이터는 기록하지 않는다.

거래유형이 존재하는 0원 행, 금액을 숫자로 변환할 수 없는 행, 날짜·잔액이 잘못된 행,
본거래/상세행 구조가 어긋난 행은 안내성 행으로 취급하지 않는다. 이러한 행은 기존 검증 오류를 유지하여
부분 저장이나 잘못된 거래 저장을 차단한다.

실행 요약의 카운터 의미는 다음과 같다.

- `parsedRowCount`: 파서가 검증된 결과 테이블에서 발견한 전체 본거래 행 수
- `skippedInformationalRowCount`: 안내성 행으로 안전하게 제외한 수
- `scrapedCount`: 안내성 행을 제외하고 정상 거래로 채택한 수
